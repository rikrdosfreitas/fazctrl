A ideia aqui é detalhar cada entidade e subentidade, separando os atributos obrigatórios (obrigatorio) e opcionais (opcionais ), já pensando na modelagem lógica que você poderá usar no banco de dados.
🧬 DOMÍNIO REPRODUTIVO

O domínio reprodutivo engloba todo o ciclo da fêmea — da aplicação de um protocolo até o parto — e os registros auxiliares como sêmen, embriões, técnicos e diagnósticos.
1. 🧫 ProtocoloReprodutivo

Define o modelo de protocolo (ex: “IATF 7 dias com BE + PG + ECP”), usado como base para múltiplas aplicações.
Tipo    Atributo    Descrição
obrigatorio    Id    Identificador único
obrigatorio    Nome    Nome do protocolo
obrigatorio    Tipo    Enum: {IATF, IA, FIV, TE}
opcionais     Descricao    Explicação geral
obrigatorio    Especie    Ex: bovino
opcionais     CategoriaAnimal    Ex: vaca, novilha
opcionais     DuracaoDias    Duração total estimada
opcionais     Ativo    Booleano

Subentidade de ProtocoloReprodutivo, define as etapas que o técnico irá seguir.

opcionais     Medicamento

opcionais     ViaAdministracao    
opcionais     Observacoes

🧩 Considerações de Modelagem

    Cada animal deve ter categoria derivada automaticamente da idade + sexo (pode ser calculada por regra).
    A genealogia (pai/mãe) deve permitir autorreferência, com controle de integridade (ex: mãe ≠ filho).
    O lote atual deve ser controlado via histórico de movimentação, permitindo rastrear quando entrou e saiu.
    O histórico de peso deve ter gatilho para atualizar PesoAtual do animal.
    O status do animal muda automaticamente quando há evento de saída, morte ou transferência.


💉 DOMÍNIO SANITÁRIO

Este domínio se divide em 6 subentidades principais:

    EventoSanitario – registro genérico de uma ação sanitária
    TipoEventoSanitario – classificação do tipo de ação (vacinação, vermifugação etc.)
    Medicamento / Vacina – catálogo de produtos aplicados
    TratamentoAnimal – tratamento contínuo para doença ou ferimento
    DoencaAnimal – diagnóstico e histórico de doenças
    ExameLaboratorial – exames clínicos e laboratoriais realizados

1. 💉 EventoSanitario

Registro individual ou em grupo de uma aplicação sanitária (vacina, vermífugo, medicamento, exame, casqueamento etc.).
Tipo    Atributo    Descrição
obrigatorio    Id    
obrigatorio    TipoEventoSanitarioId    FK → TipoEventoSanitario
opcionais     MedicamentoId    FK → Medicamento/Vacina
obrigatorio    DataAplicacao    
opcionais     HoraAplicacao    
obrigatorio    AnimalId    FK → Animal
opcionais     LoteId    se aplicado em grupo
opcionais     Dose    
opcionais     UnidadeDose    mL, g, UI, etc.
opcionais     ViaAdministracao    intramuscular, subcutânea, oral etc.
opcionais     ResponsavelId    FK → Funcionario/Técnico
opcionais     ProximaAplicacao    
opcionais     LocalAplicacao    Ex: pescoço esquerdo
opcionais     Observacoes    

    💡 Recomenda-se usar um campo “ReforçoPrevistoEmDias” em TipoEventoSanitario para calcular automaticamente “ProximaAplicacao”.

⚙️ Regras e Boas Práticas de Modelagem

    Ao criar um EventoSanitario, caso ProximaAplicacao esteja preenchida, gerar alerta automático para a agenda.

    Medicamentos vencidos ou sem estoque devem bloquear novas aplicações.

    Permitir registro em lote, aplicando o mesmo evento a vários animais.

    Tratamentos devem ser finalizados automaticamente quando ultrapassar DataFim.

    Permitir vincular exame a tratamento (ex: resultado de mastite antes/depois).

    Relatórios úteis:
        Animais com vacinas vencidas
        Histórico sanitário por animal
        Consumo de medicamentos por período
        Taxa de morbidade / mortalidade


🧩 Domínio: Produção
1. Produção de Leite

Registra a quantidade de leite produzida por animal, por ordenha ou por período, permitindo análises de produtividade.

Entidade: ProducaoLeite
Atributos:

    id (PK)
    animal_id (FK → Animal)
    data_coleta (DATE) — obrigatório
    quantidade_litros (DECIMAL(5,2)) — obrigatório
    turno (ENUM: “Manhã”, “Tarde”, “Noite”) — opcional
    ordenhador_id (FK → Pessoa ou Funcionário) — opcional
    tanque_id (FK → TanqueLeite) — opcional
    observacoes (TEXT) — opcional
    temperatura_leite (DECIMAL(4,1)) — opcional

2. Lactação

Representa o ciclo de lactação de uma fêmea, vinculado a partos e períodos de produção ativa.

Entidade: Lactacao
Atributos:

    id (PK)
    animal_id (FK → Animal)
    data_inicio (DATE) — obrigatório
    data_fim (DATE) — opcional
    numero_lactacao (INT) — obrigatório
    duracao_dias (INT) — calculado
    producao_total_litros (DECIMAL(7,2)) — calculado
    media_diaria_litros (DECIMAL(5,2)) — calculado
    estado (ENUM: “Ativa”, “Encerrada”, “Secagem”) — obrigatório

Relacionamentos:

    1 Animal → N Lactacao
    1 Lactacao → N ProducaoLeite

3. Pesagem

Registra o peso corporal dos animais para controle de crescimento e nutrição.

Entidade: Pesagem
Atributos:

    id (PK)
    animal_id (FK → Animal)
    data_pesagem (DATE) — obrigatório
    peso_kg (DECIMAL(6,2)) — obrigatório
    responsavel_id (FK → Pessoa ou Funcionário) — opcional
    condicao_corporal (DECIMAL(2,1), escala 1–5) — opcional
    observacoes (TEXT) — opcional
    equipamento_id (FK → Equipamento) — opcional

4. Ganho de Peso

Derivado das pesagens, usado para análises de desempenho.

Entidade: GanhoPeso (gerado automaticamente, mas pode ser armazenado para histórico)
Atributos:

    id (PK)
    animal_id (FK → Animal)
    data_inicio (DATE) — obrigatório
    data_fim (DATE) — obrigatório
    peso_inicial (DECIMAL(6,2)) — obrigatório
    peso_final (DECIMAL(6,2)) — obrigatório
    ganho_kg (DECIMAL(6,2)) — calculado
    dias_intervalo (INT) — calculado
    ganho_medio_diario (DECIMAL(5,2)) — calculado

5. Tanque de Leite

Controle de armazenamento e qualidade do leite.

Entidade: TanqueLeite
Atributos:

    id (PK)
    identificador (VARCHAR(50)) — obrigatório
    capacidade_litros (DECIMAL(6,2)) — obrigatório
    temperatura_atual (DECIMAL(4,1)) — opcional
    status (ENUM: “Em uso”, “Limpeza”, “Manutenção”) — obrigatório
    localizacao (VARCHAR(100)) — opcional

🔗 Principais Relacionamentos

    1 Animal → N ProducaoLeite
    1 Animal → N Pesagem
    1 Animal → N Lactacao
    1 Lactacao → N ProducaoLeite
    1 TanqueLeite → N ProducaoLeite

Atributos:

    1 PlanoNutricao → N AlimentacaoAnimal

6. Controle de Estoque Nutricional

Gerencia o estoque de insumos alimentares.

Entidade: EstoqueNutricao
Atributos:

    id (PK)
    produto_tipo (ENUM: “Ração”, “Suplemento”) — obrigatório
    produto_id (FK → Racao ou Suplemento)
    quantidade_atual_kg (DECIMAL(8,2)) — obrigatório
    data_atualizacao (DATE) — obrigatório
    local_armazenamento (VARCHAR(100)) — opcional
    responsavel_id (FK → Pessoa ou Funcionário) — opcional
